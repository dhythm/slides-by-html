# スライドレイアウトパターン分析書

## 1. 本書の位置づけ

本書は、スライドレイアウトパターンの設計にあたり、3 つの情報源を横断的に分析した結果をまとめたものである。

分析対象は以下の 3 ソースである。

| ソース | 形式 | パターン数 | 特徴 |
|--------|------|:--------:|------|
| business_plan_reveal.pdf | Reveal.js サンプルデッキ（20 枚） | 約 18 種の複合レイアウト | 事業計画に特化。1 スライドに複数の基本構図を組み合わせた複合レイアウトが多い |
| ワークワンダース レイアウトパターン集.pdf | Marp スライド（80 ページ） | 67 種 | 10 カテゴリに分類された汎用パターンカタログ。基本構図を網羅的に収録 |
| 追加フレームワーク（ユーザー提供） | テキスト要件 | 54 種（A〜H の 8 カテゴリ） | 実務観点での分類。Microsoft、Canva、Beautiful.ai 等の分析を踏まえた体系 |

|-----------|------|-------|
| KPI + グラフ | 数値タイル行 + グラフ | p.1 ヒーロー、p.3 サマリー、p.14 ユニットエコノミクス |
| グラフ + 表 | メイングラフ + 補足テーブル | p.15 3 か年計画 |
| カード + 図解 | カードグリッド + ファネル or マトリクス | p.10 GTM、p.11 競合 |
| 引用 + 数値 | 顧客の声 + Before/After 数値 | p.13 顧客事例 |
| テキスト + UI モック | 説明文 + 画面キャプチャ | p.6 ソリューション |
| 人物 + 計画 | チームカード + 採用計画バー | p.16 組織と採用 |

### 5.2 複合レイアウトの設計方針

複合レイアウトは独立したパターンとして定義するのではなく、基本パターンの組み合わせとして構成するのが望ましい。理由は以下である。

- 基本パターンの組み合わせ数は膨大であり、すべてを個別定義するのは非現実的である
- 基本パターンが堅牢であれば、組み合わせは CSS Grid や Flexbox で柔軟に実現できる
- 組み合わせルールを定義する方が、個別テンプレートを量産するより保守性が高い

ただし、特に頻出する組み合わせ（KPI + グラフ、引用 + 数値など）は、組み合わせテンプレートとして提供すると実用的である。

## 6. バリエーション管理の方針

WW 67 種の中には、同一の構図に対して複数のバリエーションを持つものがある。これらは別パターンではなく、1 つのパターン内のオプションとして管理すべきである。

| 親パターン | バリエーション | 切り替え方法 |
|----------|-------------|-----------|
| 2 カラム | 均等 / 6:4 / 4:6 | CSS 変数またはクラス |
| 3 カラム | 均等 / アイコン付き | コンテンツの有無で自動判定 |
| ステップ型 | 縦 / 横 | クラスで方向指定 |
| タイムライン | 縦 / 横 / 蛇行 | クラスで方向指定 |
| プロセスフロー | 基本 / 矢印（chevron） | クラスでスタイル指定 |
| 表 | シンプル / ストライプ / 強調 | クラスでスタイル指定 |
| 棒グラフ | 縦 / 横 / 積み上げ / 100% | データ属性で指定 |
| セクション扉 | 基本 / 番号付き / アイコン付き | コンテンツの有無で自動判定 |

## 7. 結論

### 7.1 コア 28 パターンの妥当性

3 ソースの横断分析の結果、コア 28 パターンは以下の点で妥当である。

- WW 67 種のカバー率 82%。不足分は複合構図や特化構図であり、基本構図は網羅されている
- Reveal PDF の全 20 スライドを、コア 28 の組み合わせで再現可能である
- フレームワーク 54 種のうち、高頻度で使用されるものはすべてコア 28 に含まれている

### 7.2 拡張候補の優先順位

コア 28 に追加する場合の優先順位は以下の通りである。

| 優先度 | パターン | 理由 |
|:-----:|---------|------|
| 高 | 円グラフ / ドーナツ | Reveal PDF で 2 回使用。構成比説明は頻出 |
| 高 | CTA / 次のアクション | 営業資料、提案書で必須 |
| 高 | 顧客の声 / ロゴ実績 | 信頼形成に不可欠。Reveal PDF で 2 回使用 |
| 中 | 縦タイムライン | モバイル対応や長い時系列で有用 |
| 中 | SWOT | 戦略系資料で定番 |
| 中 | 全画面画像 + 短文 | ビジュアル訴求に有効 |
| 低 | ガントチャート | プロジェクト管理特化 |
| 低 | 地図 / 市場分布 | 地域展開特化。実装難度が高い |
| 低 | デバイスモックアップ | SaaS / アプリ紹介特化 |

### 7.3 装飾コンポーネントとして別管理すべきもの

以下は、レイアウトパターンではなく、共通デザインルールの装飾コンポーネントとして管理する。

- パネル（基本、アイコン付き、ボーダーアクセント、グラデーション）
- カード（情報カード、統計カード、人物カード）
- タグ / バッジ
- 数値ハイライト（大数字 + 補足）
- 引用ブロック（左ボーダー + イタリック）
